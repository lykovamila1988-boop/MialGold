# Оля: Контекстный анализ трендов и цепочки (Chain_ID)

## 📊 Обзор

Оля (исследователь трендов) теперь отслеживает **контекст каждого анализа** через систему **chain_id**. Это позволяет:

1. **Корректировать анализ** в зависимости от того, кто запрашивает (user vs marina vs victoria)
2. **Логировать всю историю** тренд-исследований в аналитику
3. **Измерять влияние** контекста на результаты
4. **Сохранять полную цепочку** для отладки и оптимизации

---

## 🔄 Как работает контекст

### До (без контекста)

```
User: "/тренды"
  ↓
Oля анализирует вслепую
  ├─ Не знает: это для быстрой идеи или долгосрочной стратегии?
  ├─ Не знает: это для поста Марины или переделки Виктории?
  ├─ Не знает: это срочно нужно (завтра) или есть время?
  └─ Результат: общий анализ, может быть не релевантный
```

### Теперь (с контекстом)

```
Marina: "/тренды" [from: marina, chain_id: post_2026_06_08_1]
  ↓
Oля анализирует с контекстом
  ├─ Знает: от Marina → узкий анализ под пост
  ├─ Знает: chain_id: post_2026_06_08_1 → может отслеживать историю
  ├─ Знает: это срочно (поток работы) → фокусируется на быстро вирусящихся темах
  └─ Результат: конкретные 2-3 угла под пост, хорошо таргетированы

Victoria: "/тренды" [from: victoria, chain_id: post_2026_06_08_1]
  ↓
Oля анализирует по-другому
  ├─ Знает: от Victoria → фокус на аутентичности голоса
  ├─ Знает: это редактура, а не новый контент → углубленный анализ паттернов
  └─ Результат: тренды которые резонируют с брендом Людмилы
```

## 🎯 Разные подходы в зависимости от `from_agent`

| from_agent | Глубина | Охват | Временной горизонт | Фокус |
|---|---|---|---|---|
| **user** | FULL | 5-10 тем + стратегия | неделя-месяц | Все углы, паттерны |
| **marina** | NARROW | 2-3 угла под пост | час-день (срочно) | Только релевантное |
| **victoria** | VOICE | 3-5 вариантов | долгоиграющие | Аутентичность, голос |
| **vasya** | TIMING | быстро вирусящиеся | час, день (микротренды) | КОГДА и КАКОЙ формат |
| **rita** | VISUAL | визуальные тренды | неделя | Палитра, стиль, эффекты |

## 📝 Логирование цепочки

### Где хранятся логи

```
E:\MILA GOLD\
  └─ MILA-BUSINESS/05-analytics/trends-chains/
     ├─ post_2026_06_08_1.log     ← один пост, прошёл через несколько агентов
     ├─ content_v2.log             ← одна задача, несколько итераций
     ├─ direct_trend_20260608_143022.log  ← прямой запрос от user
     └─ ...
```

### Формат лога

Каждая строка — один вызов функции в цепочке:

```json
{"timestamp": "2026-06-08T14:30:15.123456", "from_agent": "marina", "analysis_type": "web_search_serpapi", "result_length": 1250, "result_preview": "query=выбор партнёра, results=10"}
{"timestamp": "2026-06-08T14:31:02.654321", "from_agent": "marina", "analysis_type": "monitor_competitors", "result_length": 3400, "result_preview": "checked=8 accounts, total_len=3400"}
{"timestamp": "2026-06-08T14:32:45.987654", "from_agent": "victoria", "analysis_type": "context_analysis", "result_length": 2100, "result_preview": "📊 КОНТЕКСТ АНАЛИЗА ТРЕНДОВ\n\n┌─ Запрос пришёл ──..."}
```

### Интерпретация

- **timestamp** — когда была выполнена операция
- **from_agent** — кто запросил анализ (переходит от marina → victoria → vasya)
- **analysis_type** — какой инструмент использовался (web_search, monitor_competitors и т.д.)
- **result_length** — размер результата (показывает объём данных)
- **result_preview** — первые 200 символов (для быстрого просмотра)

## 📊 Как читать логи (аналитика)

### Быстрая проверка цепочки

```bash

# Фильтр: сколько результатов вернула web_search?
grep "analysis_type.*web_search" MILA-BUSINESS/05-analytics/trends-chains/*.log | head -20
```

### Анализ производительности цепочки

```python
import json
from pathlib import Path

def analyze_trends_chain(chain_id):
    """Анализ одной цепочки"""
    log_file = Path(f"MILA-BUSINESS/05-analytics/trends-chains/{chain_id}.log")
    
    if not log_file.exists():
        return f"Логи для {chain_id} не найдены"
    
    entries = []
    for line in log_file.read_text(encoding="utf-8").split("\n"):
        if line.strip():
            entries.append(json.loads(line))
    
    # Подсчитываем
    by_agent = {}
    by_type = {}
    
    for entry in entries:
        agent = entry["from_agent"]
        atype = entry["analysis_type"]
        
        by_agent[agent] = by_agent.get(agent, 0) + 1
        by_type[atype] = by_type.get(atype, 0) + 1
    
    print(f"Цепочка: {chain_id}")
    print(f"Всего операций: {len(entries)}")
    print(f"По агентам: {by_agent}")
    print(f"По типам анализа: {by_type}")

analyze_trends_chain("post_2026_06_08_1")
```

## 🛠️ Как это работает внутри

### Код: Извлечение контекста

```python
# olya.py - когда приходит запрос
def _extract_context(inp: dict) -> dict:
    from_agent = inp.get("from_agent", "user")
    chain_id = inp.get("chain_id", "")
    
    return {
        "from_agent": from_agent,
        "chain_id": chain_id or f"direct_trend_{datetime.now().strftime('%Y%m%d_%H%M%S')}",
        "timestamp": datetime.now().isoformat(),
        "has_context": bool(chain_id)
    }
```

### Код: Логирование анализа

```python
def _log_trend_analysis(chain_id: str, from_agent: str, analysis_type: str, result: str) -> None:
    """Логирует каждый вызов функции"""
    log_file = TRENDS_LOG_DIR / f"{chain_id}.log"
    log_entry = {
        "timestamp": datetime.now().isoformat(),
        "from_agent": from_agent,
        "analysis_type": analysis_type,
        "result_length": len(result),
        "result_preview": result[:200]
    }
    
    with open(log_file, "a", encoding="utf-8") as f:
        f.write(json.dumps(log_entry, ensure_ascii=False) + "\n")
```

### Код: Использование контекста в web_search

