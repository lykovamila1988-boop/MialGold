## Технические детали (минимум)

### Как работает контекст

1. Браузер отправляет запрос с `from_agent` и `chain_id`
2. Oля извлекает контекст функцией `_extract_context(inp)`
3. Выбирает подход в зависимости от `from_agent`
4. Логирует результат в `trends-chains/{chain_id}.log`

### Где находится код

```
mila-office/olya.py
├─ _extract_context()      ← извлечение контекста
├─ _log_trend_analysis()   ← логирование
├─ get_trend_context()     ← демонстрация контекста
└─ handle()                ← dispatcher (передаёт контекст)
```

### Что логируется

```json
{
    "timestamp": "2026-06-08T14:30:15.123456",
    "from_agent": "marina",
    "analysis_type": "web_search",
    "result_length": 1250,
    "result_preview": "query=..., results=10"
}
```

## Частые вопросы

### Вопрос: Где логи?

**Ответ:** `MILA-BUSINESS/05-analytics/trends-chains/{chain_id}.log`

```bash
cat "e:\MILA GOLD\MILA-BUSINESS\05-analytics\trends-chains\post_2026_06_08_1.log"
```

### Вопрос: Как Oля знает подход?

**Ответ:** Смотрит на `from_agent`:
- `from_agent=user` → FULL подход
- `from_agent=marina` → NARROW подход
- `from_agent=victoria` → VOICE подход
- `from_agent=vasya` → TIMING подход

### Вопрос: Почему результаты разные?

**Ответ:** Потому что контекст меняет:
1. Какие тренды Oля ищет
2. Сколько результатов берёт
3. Какую глубину анализирует
4. Как логирует результаты

### Вопрос: Я хочу свой подход

**Ответ:** Добавь в `analysis_approach` словарь (olya.py, линия ~175):

```python
analysis_approach = {
    "user": {...},
    "marina": {...},
    "victoria": {...},
    "vasya": {...},
    "твой_агент": {  # ← добавь сюда
        "depth": "CUSTOM",
        "scope": "твой охват",
        "timeline": "твой горизонт"
    }
}
```
