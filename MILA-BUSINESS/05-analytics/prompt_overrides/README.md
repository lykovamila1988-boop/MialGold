# prompt_overrides/ — слой самообучения офиса

Сюда **Стас** (офис-менеджер) пишет data-driven улучшения промптов агентов.
`base.compose_system(key, system)` читает `<key>.md` динамически при каждом запуске
агента и подмешивает к его системному промпту — код агентов не меняется, эффект
обратим (удалил файл → откатил улучшение).

Файлы по ключу агента: `marina.md`, `victoria.md`, `olya.md`, `lera.md`, …

## Стратегия (важно — иначе промпт распухнет и начнёт противоречить себе)

НЕ append-only. Каждый `<key>.md` — структурированный файл с датированными
выводами по темам. При новом выводе по той же теме старый **вытесняется**, а не
дописывается. Активных выводов — не больше ~7 (самые свежие/сильные). Полная
история изменений уходит в `improvement_log.md` (а не в промпт агента).

Формат одного вывода:
```
## <тема> — обновлено YYYY-MM-DD
<краткий вывод с цифрами: что делать и почему, на основе данных>
```
