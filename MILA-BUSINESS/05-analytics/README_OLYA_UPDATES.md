# Обновление Oли: Контекстный анализ трендов с Chain_ID отслеживанием

## 🎯 Что было сделано

Oля (исследователь трендов) теперь работает **с контекстом цепочки** (chain_id). Это означает:

1. **Контекст знаний** — Oля знает КТО её запрашивает (user vs marina vs victoria vs vasya)
2. **Адаптивный анализ** — меняет глубину в зависимости от от_агента
3. **Полное логирование** — каждый анализ записывается в лог цепочки
4. **Аналитика** — можно видеть как контекст влияет на результаты

## 📝 Логирование: как читать логи

### Структура лога

Каждая строка — JSON с одной операцией:

```json
{
    "timestamp": "2026-06-08T14:30:15.123456",
    "from_agent": "marina",
    "analysis_type": "web_search",
    "result_length": 1250,
    "result_preview": "query=выбор партнёра, results=10"
}
```

### Типы операций (analysis_type)

- `web_search_serpapi` — поиск через SerpApi (платный, точнее)
- `web_search_ddg` — поиск через DuckDuckGo (бесплатный, фолбэк)
- `web_search_error` — ошибка поиска
- `monitor_competitors` — мониторинг конкурентов
- `context_analysis` — вызов get_trend_context
- `trends_error`, `monitor_error` — ошибки

### Пример анализа

```bash
# Найти все операции для конкретного поста
cat MILA-BUSINESS/05-analytics/trends-chains/post_2026_06_08_1.log

# Выходит:
# [14:30:15] marina → web_search_serpapi: 1250 символов
# [14:31:02] marina → monitor_competitors: 2100 символов
# [14:35:42] victoria → context_analysis: 2500 символов
# [14:36:10] victoria → web_search_serpapi: 1500 символов

# Видно: Marina работала 2 минуты, Victoria добавила 50 сек
# Разные типы анализа → разные глубины
```
