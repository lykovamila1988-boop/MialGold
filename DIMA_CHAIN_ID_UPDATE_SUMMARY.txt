================================================================================
KEY CONCEPT: Chain_ID
================================================================================

A chain_id is a unique identifier for a complete financial cycle:
- Expense (what was spent)
- Output (what was created)
- Revenue (what came back)
- ROI (how efficient was this?)

Examples:
  content-post-20260608     → Photo post spent $28.50 → earned $194 (580% ROI)
  reel-production-20260610  → Reel spent $120 → earned $314 (162% ROI)
  paid-ads-instagram        → Ads spent $275 → earned $614 (123% ROI)

================================================================================
HOW CONTEXT AFFECTS DECISIONS
================================================================================

Financial decision is automatic once you know the ROI and have context:

  ROI >= 300%    → "Inвестируй больше" (increase budget, scale up)
  ROI 100-299%   → "Продолжай в том же духе" (maintain current level)
  ROI 0-99%      → "Нужна оптимизация" (cut costs or increase conversion)
  ROI < 0%       → "Срочные правки" (pause immediately)

Example decision flow:
  User: "Что мне делать в июле? Где потратить деньги?"
  Дима: Shows all chains with ROI, sorted worst-first
  Result: "Контент-посты работают (580% ROI), рилы хорошие (162%), реклама
           работает но слабо (123%). Приоритет: масштабировать контент-посты,
           оптимизировать рекламу, поддерживать рилы."

================================================================================
FUNCTIONS AND USAGE
================================================================================

1. get_expenses_by_chain(chain_id: str) → JSON
   Input:  "reel-production-20260610"
   Output: List of all expenses for that chain (date, category, amount, note)
   Use:    Understand where money went for this chain

2. get_revenue_by_chain(chain_id: str) → JSON
   Input:  "reel-production-20260610"
   Output: List of all revenue attributed to that chain
   Use:    See which customers bought because of this chain

3. calculate_chain_roi(chain_id: str) → JSON
   Input:  "reel-production-20260610"
   Output: {
             chain_id, total_expense, total_revenue, net_profit, roi_percent,
             verdict (✓/⚠/✗), financial_decision (action to take),
             recommendation (context + reasoning)
           }
   Use:    Get ROI with context and financial decision

4. get_all_chains_summary() → JSON
   Input:  (none)
   Output: {
             total_chains, total_expense, total_revenue, total_net_profit,
             chains_by_roi (sorted), action_items (urgent problems)
           }
   Use:    See the full picture - which chains work, which are problems

SUMMARY
================================================================================

Дима now understands the CONTEXT of financial flows. Every expense is tied to
a revenue source. Every decision is based on ROI and what that ROI means for
next month's budget.

Key innovation: Financial decisions are automatic and data-driven
  - See ROI → automatic verdict (excellent/good/weak/losing)
  - See verdict → automatic decision (increase/maintain/optimize/pause)
  - See decision → allocate budget accordingly

This transforms financial management from "How much did we make?" to "Where
should we spend more?" — a fundamentally different (and better) conversation.
