================================================================================
CHAIN_ID FORMAT
================================================================================

Instagram cross-post:
  ig_20260608_a1b2c3
  └─ Prefix: "ig_" (Instagram source)
  └─ Date: "20260608" (YYYYMMDD)
  └─ UUID: "a1b2c3" (first 6 chars of UUID)

Telegram-only post:
  tg_a1b2c3d4
  └─ Prefix: "tg_" (Telegram-only)
  └─ UUID: "a1b2c3d4" (first 8 chars of UUID)

Purpose:
  - Unique across both platforms
  - Tracks which posts came from where
  - Links Instagram version to Telegram version in analytics

================================================================================
METADATA STRUCTURE
================================================================================

Every message in queue carries metadata:

{
  "channel_id": "-1001234567890",           # Telegram channel ID
  "chat_id": "987654321",                   # User ID (for DM)
  "chain_id": "ig_20260608_a1b2c3",         # Cross-post link
  "source": "instagram" or "telegram",      # Where it came from
  "content_type": "insight",                # insight/case/practice/offer/diagnostic/welcome
  "created_at": "2026-06-08T10:00:00Z"      # Timestamp
}

================================================================================
INTEGRATION WITH OTHER AGENTS
================================================================================

Marina (Instagram):
  - Creates posts with chain_id
  - Puts in "instagram_comments" queue with metadata
  - Returns chain_id for downstream agents

Vasya (Scheduler/Publisher):
  - Takes from queue, publishes to Instagram
  - PRESERVES chain_id in metadata
  - Records in memory.record_published(..., extra={"chain_id": "..."})

Tyoma (Telegram):
  - Finds Instagram posts by chain_id
  - Adapts voice/content for Telegram
  - Puts in "telegram" queue with SAME chain_id
  - Can track cross-post performance in analytics

KEY CONCEPTS TO REMEMBER
================================================================================

1. chain_id = Sync key across platforms (Instagram ↔ Telegram)

2. Unified queue = single message.queue_message() called by all agents
   (not separate Instagram API + Telegram API)

3. Metadata = how agents communicate:
   - chain_id tells Tyoma which Instagram post to adapt
   - content_type helps with analytics & decision-making
   - source tracks where the content originated

4. Adaptation = Telegram version is different from Instagram:
   - Shorter ≠ longer (Telegram can have longer text per message)
   - No emoji ≠ emoji-rich (Instagram is visual, Telegram is text+emoji)
   - No links ≠ full links (Instagram bans links to avoid spam, Telegram encourages CTAs)

5. Two sources of truth:
   - memory.published: Instagram metrics (48h after publish)
   - memory.message_queue: Telegram metrics (sent status + user responses)
   Both linked by chain_id for cross-platform analytics

QUESTIONS & NEXT STEPS
================================================================================

Q: What if Marina doesn't create chain_id?
A: Tyoma creates one when putting message in queue (tg_ prefix)
   But then it won't link to Instagram. Better: Marina always creates.

Q: Can I cross-post everything to Telegram?
A: No. Use decision tree in GUIDE:
   - Instagram ONLY: political, personal, exclusive Instagram content
   - Instagram + Telegram: psychology, patterns, diagnostics, offers
   - Telegram ONLY: welcome, fast responses, subscriber-only content

Q: How does Vasya know to preserve chain_id?
A: See Vasya's code in vasya.py:
   memory.record_published(..., extra={"chain_id": chain_id})
   Must be added to Vasya's workflow.

Q: Can I measure success of cross-posting?
A: Yes! Both versions share chain_id in metadata.
   Compare Instagram reach (48h) vs Telegram engagement (immediate CTAs).
   See GUIDE: compare_cross_post_performance() example.
