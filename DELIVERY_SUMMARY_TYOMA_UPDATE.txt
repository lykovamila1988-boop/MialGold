================================================================================
CHAIN_ID CONCEPT EXPLAINED
================================================================================

What is chain_id?
  A unique identifier that links content across platforms.

  Example:
  - Instagram post created by Marina
  - Published by Vasya with chain_id="ig_20260608_a1b2c3"
  - Adapted for Telegram by Tyoma with same chain_id
  - Both versions tracked in analytics with same chain_id

Format:
  Instagram source:  "ig_20260608_a1b2c3"  (prefix + date + uuid)
  Telegram-only:     "tg_a1b2c3d4"          (prefix + uuid)

How it works:
  1. Marina creates post → assigns chain_id
  2. Vasya publishes to Instagram → preserves chain_id
  3. Tyoma finds by chain_id → adapts for Telegram → posts with same chain_id
  4. Analytics compares both versions using chain_id

================================================================================
KEY INTEGRATION POINTS
================================================================================

With Marina (Instagram Agent):
  - Creates content
  - Generates chain_id
  - Puts in "instagram_comments" queue with metadata

With Vasya (Scheduler/Publisher):
  - Takes from queue
  - Publishes to Instagram
  - MUST preserve chain_id in metadata
  - Records in memory.record_published(..., extra={"chain_id": ...})

With Tyoma (Telegram Agent):
  - Finds Instagram posts by chain_id
  - Adapts voice/content for Telegram
  - Puts in "telegram" queue with SAME chain_id
  - Can track cross-post performance

With Memory (memory.py):
  - No changes needed
  - Already supports metadata in messages
  - chain_id lives in metadata

USAGE SCENARIOS
================================================================================

SCENARIO 1: Automatic Cross-Posting
──────────────────────────────────
User: /синхро
Tyoma:
  1. Scans Instagram queue for new posts with chain_id
  2. For each post:
     a. Calls get_cross_post_context(chain_id)
     b. Retrieves original text
     c. Adapts for Telegram (add links, emoji, longer text)
     d. Calls send_to_queue(..., chain_id=original_id, ...)
  3. Reports status

SCENARIO 2: Manual Cross-Post
──────────────────────────────
User: /кросс ig_20260608_a1b2c3
Tyoma:
  1. Calls get_cross_post_context("ig_20260608_a1b2c3")
  2. Shows original Instagram text
  3. Prompts user to confirm or edit Telegram version
  4. Calls send_to_queue(..., chain_id="ig_20260608_a1b2c3", ...)

SCENARIO 3: Telegram-Only Content
──────────────────────────────────
User: /цепочка
Tyoma:
  1. Creates welcome message
  2. Calls send_to_queue(text, channel_id=..., content_type="welcome")
  3. chain_id auto-generated as "tg_XXXXX" (Telegram-only)
  4. No Instagram link

SCENARIO 4: Check Queue Status
───────────────────────────────
User: /очередь
Tyoma:
  1. Calls list_pending_telegram(limit=20)
  2. Shows all pending messages
  3. Displays: text preview, status, content_type, chain_id
  4. User can see what's waiting to be sent

TELEGRAM-SPECIFIC ADAPTATIONS

  Instagram: Shorter (algorithm favors)

EMOJI USAGE
  Instagram: Minimal (too many looks spammy)
  Telegram: Emoji-rich (helps scanning & engagement)

  Instagram: "Что происходит: codependency, обида, потеря границ"
  Telegram: "💔 Codependency
             🔴 Обида и resentment
             🚫 Потеря границ"

CALLS TO ACTION
  Instagram: "ХОЧУ" in comments
  Telegram: Direct links + "напиши ХОЧУ" for personal response

================================================================================
METADATA STRUCTURE
================================================================================

Every message in unified queue carries metadata:

{
  "channel_id": "-1001234567890",           # Telegram channel ID
  "chat_id": "987654321",                   # User ID (for DM)
  "chain_id": "ig_20260608_a1b2c3",         # SYNC KEY (cross-post link)
  "source": "instagram" | "telegram",       # Where it came from
  "content_type": "insight",                # insight/case/practice/offer/diagnostic/welcome
  "created_at": "2026-06-08T10:00:00Z"      # Timestamp
}

Why metadata?
  - Allows agents to communicate without hard dependencies
  - chain_id links Instagram ↔ Telegram versions
  - content_type helps with analytics & filtering
  - source tracks origin
  - created_at enables lifecycle tracking
