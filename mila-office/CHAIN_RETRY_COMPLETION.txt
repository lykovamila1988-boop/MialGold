================================================================================
FEATURES IMPLEMENTED
================================================================================

1. RETRY_CHAIN (Restart from failed agent)
   - Configurable max retries (default: 3)
   - Automatic retry count tracking
   - Reset downstream agents to pending
   - Integration with error_monitor for alerts
   - Telegram notification on critical failure

2. ESCALATE_CHAIN (Switch to different agent)
   - Redirect to more capable agent
   - Skip remaining agents in original flow
   - Dynamic agents list update
   - Full audit trail

3. SPLIT_CHAIN (Parallel processing)
   - Send to multiple agents simultaneously
   - Independent tracking per branch
   - Branch metadata storage

4. MERGE_RESULTS (Combine parallel results)
   - "union" strategy - collect all successful results
   - "consensus" strategy - pick highest confidence
   - "first_success" strategy - first non-error result
   - Graceful error handling

5. STATE & LOGGING
   - In-memory chain state (thread-safe with RLock)
   - Structured JSON event logging (JSONL)
   - Separate retry attempt log
   - Text log for debugging
   - JSON export/import capability
   - Query functions for history & stats

6. ERROR HANDLING
   - Integration with error_monitor.py
   - Automatic Telegram alerts on critical failures
   - Structured error context
   - Full traceback storage
   - Configurable alert levels

IMPLEMENTATION SUMMARY
================================================================================

ARCHITECTURE:

  _chains: Dict[chain_id -> chain_object]
    Protected by: _chains_lock (RLock)

  Chain Object Structure:
    - id: string
    - status: ChainStatus enum (RUNNING, SUCCESS, FAILED, RETRYING, etc.)
    - agents: List[string]
    - nodes: Dict[agent -> ChainNode]
    - retry_count: int
    - created_at: ISO timestamp
    - completed_at: ISO timestamp (optional)
    - context: Dict (user-provided metadata)
    - history: List[ChainEvent]
    - split_branches: Dict (for parallel execution)
    - merged_results: Dict (for merged output)

  ChainNode Structure:
    - agent_key: string
    - status: string (pending/running/success/failed/skipped)
    - reply: string (optional)
    - error: string (optional)
    - retry_count: int
    - started_at: ISO timestamp (optional)
    - completed_at: ISO timestamp (optional)
    - duration_seconds: float (optional)

  ChainEvent Structure:
    - timestamp: ISO timestamp
    - event_type: string (start/node_done/node_failed/retry/escalate/split/merge/success/cancel)
    - chain_id: string
    - agent_key: string
    - details: Dict

THREAD SAFETY:
  - All state mutations protected by RLock
  - Safe for concurrent access from multiple threads
  - No race conditions on _chains dictionary

LOGGING:
  - logs/chain_events.jsonl - All events (append-only)
  - logs/chain_retries.jsonl - Retry attempts only
  - logs/chain_retry.log - Text log with timestamps
  - Integration with error_monitor for critical alerts

API REFERENCE (SUMMARY)
================================================================================

PRIMARY FUNCTIONS:

1. create_chain(chain_id, agents, context=None)
   Create a new workflow chain

2. update_node_status(chain_id, agent_key, status, reply=None, error=None)
   Update agent progress (running/success/failed/skipped)

3. retry_chain(chain_id, failed_agent, reason, max_retries=3)
   Restart from failed agent with automatic limit enforcement

4. escalate_chain(chain_id, new_agent, reason="")
   Switch to different agent (for complex/stuck tasks)

5. split_chain(chain_id, to_agents, context=None)
   Send to multiple agents in parallel

6. merge_results(chain_id, results, merge_strategy="union")
   Combine parallel results (union/consensus/first_success)

7. complete_chain(chain_id, final_result=None)
   Mark workflow as successfully completed

8. cancel_chain(chain_id, reason="")
   Stop the workflow

QUERY FUNCTIONS:

- get_chain(chain_id) -> Chain object or None
- get_chain_history(chain_id) -> List of events
- get_chain_stats(hours=24) -> Aggregated statistics
- get_all_chains() -> All active chains

MAINTENANCE:

- clear_old_chains(hours=24) -> Delete old chains from memory
- export_chain_to_json(chain_id, filepath=None) -> Save to JSON
- load_chain_from_json(filepath) -> Restore from JSON
