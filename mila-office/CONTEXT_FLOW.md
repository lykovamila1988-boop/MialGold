# 🔄 CONTEXT FLOW — Визуализация потока контекста

## ДИАГРАММА: Полный цикл с контекстом

```
┌──────────────────────────────────────────────────────────────┐
│ 1. ПОЛЬЗОВАТЕЛЬ ОТПРАВЛЯЕТ МАРИНЕ                             │
└──────────────────────────────────────────────────────────────┘

  Browser:
  POST /api/chat {
    agent: "marina",
    message: "Напиши пост",
    from_agent: "user",           ← Отправитель: пользователь
    chain_id: "wf_123"            ← Цепочка обработки
  }
                ↓
┌──────────────────────────────────────────────────────────────┐
│ 2. СЕРВЕР СОЗДАЁТ ЗАДАНИЕ С КОНТЕКСТОМ                       │
└──────────────────────────────────────────────────────────────┘

  routes.py /api/chat:
  job = {
    id: "job_abc123",
    agent_key: "marina",
    from_agent: "user",           ← От кого
    to_agent: null,               ← Кому (null = дальше в цепочку)
    chain_id: "wf_123",           ← ID цепочки
    status: "pending"
  }
                ↓
┌──────────────────────────────────────────────────────────────┐
│ 3. МАРИНА ОБРАБАТЫВАЕТ И ВИДИТ КОНТЕКСТ                       │
└──────────────────────────────────────────────────────────────┘

  Входящее сообщение:
  [from: user] [chain_id: wf_123]
  Напиши пост про выбор

  marina.py обрабатывает:
  ✓ Знает что от user (не от другого агента)
  ✓ Знает что это часть wf_123 (отслеживаемая работа)
  ✓ Может написать более контекстный ответ

  Ответ Марины:
  Пост готов! [VERDICT: ready_next] [→ victoria]
                ↓
┌──────────────────────────────────────────────────────────────┐
│ 4. СЕРВЕР ОБРАБАТЫВАЕТ РЕЗУЛЬТАТ И ПОДГОТАВЛИВАЕТ КОНТЕКСТ    │
└──────────────────────────────────────────────────────────────┘

  routes.py /api/result:
  
  job ← message_handler.process_agent_response()
  
  {
    agent_key: "marina",
    from_agent: "user",           ← Помним кто отправил
    chain_id: "wf_123",           ← Помним цепочку
    reply: "Пост готов!",
    verdict: "ready_next",
    next_agent: "victoria",
    
    chain_context: {
      current_agent: "marina",
      from_agent: "user",
      original_to_agent: null,
      chain_id: "wf_123"          ← Полный контекст для следующего!
    }
  }
                ↓
┌──────────────────────────────────────────────────────────────┐
│ 5. БРАУЗЕР ВИДИТ NEXT_AGENT И ПЕРЕХОДИТ НА ВИКТОРИЮ           │
└──────────────────────────────────────────────────────────────┘

  JavaScript (ui):
  if (result.next_agent && result.next_agent !== current) {
    switchAgent(result.next_agent);  // → переходим на victoria
  }
  
  // Подготавливаем вызов Виктории с сохранением контекста
  chain_context = result.chain_context;
                ↓
┌──────────────────────────────────────────────────────────────┐
│ 6. БРАУЗЕР ОТПРАВЛЯЕТ ВИКТОРИЮ С ПОЛНЫМ КОНТЕКСТОМ            │
└──────────────────────────────────────────────────────────────┘

  Browser:
  POST /api/chat {
    agent: "victoria",
    message: "Пост от Марины: ...",
    from_agent: "marina",         ← ИЗМЕНИЛОСЬ! Теперь от Марины
    chain_id: "wf_123",           ← Сохранилась цепочка
    to_agent: null                ← Может быть null (дальше в цепочку)
  }
                ↓
┌──────────────────────────────────────────────────────────────┐
│ 7. ВИКТОРИЯ ВИДИТ ПОЛНЫЙ КОНТЕКСТ                             │
└──────────────────────────────────────────────────────────────┘

  Входящее сообщение:
  [from: marina] [chain_id: wf_123]
  Пост от Марины: ...

  victoria.py обрабатывает:
  ✓ Знает что от marina (не от user!)
  ✓ Знает что это часть wf_123
  ✓ Понимает: "Марина уже обработала, я редактирую"
  ✓ Может сделать контекстную редактуру

  Ответ Виктории:
  [ДОКУМЕНТ]Готовый текст[/ДОКУМЕНТ]
  [VERDICT: done] [→ vasya]
                ↓
┌──────────────────────────────────────────────────────────────┘
│ ЦЕПОЧКА ПРОДОЛЖАЕТСЯ ИЛИ ЗАВЕРШАЕТСЯ                          │
└──────────────────────────────────────────────────────────────┘

  Если [VERDICT: done]:
    ✓ Работа завершена
    ✓ Результат может быть скачан
    ✓ Цепочка замкнута

  Если [→ vasya]:
    ✓ Переходим на Васю
    ✓ Виктория становится from_agent
    ✓ Контекст цепочки сохраняется
    ✓ Вася видит всю историю
```

## 🎭 СЦЕНАРИИ ИСПОЛЬЗОВАНИЯ

### Сценарий 1: Линейная цепочка

```
User
  ↓ [from: user, chain_id: wf_1]
Marina
  ├─ Видит: от user, часть wf_1
  └─ [from: marina]
  ↓
Victoria
  ├─ Видит: от marina, часть wf_1
  └─ [from: victoria]
  ↓
Vasya
  ├─ Видит: от victoria, часть wf_1
  └─ [VERDICT: done]
  
✅ Результат: прошёл через 3 агентов, все знали контекст
```

### Сценарий 2: Деделегирование

```
User → [to: rita] "Пост + визуалы"
  ↓
Marina
  ├─ Видит: от user, to: rita
  ├─ Пишет пост
  └─ [to: rita]
  ↓
Victoria
  ├─ Видит: от marina, to: rita
  ├─ Редактирует
  └─ [to: rita]  ← Прокидывает дальше!
  ↓
Rita (получает адресованное ей!)
  ├─ Видит: от victoria, to: rita
  ├─ Знает что это специально для неё
  ├─ Создаёт визуалы
  └─ [VERDICT: done]

✅ Результат: Rita знала что задача специально для неё
```

### Сценарий 3: Параллельные задачи с одной цепочкой

```
User [chain_id: wf_2] → 3 параллельные задачи:

  1) Marina (пост)
     [from: user, chain_id: wf_2]
     → Victoria (редактура)
     [from: marina, chain_id: wf_2]
     
  2) Rita (визуалы)
     [from: user, chain_id: wf_2]
     → (независимо работает)
     
  3) Tyoma (Telegram)
     [from: user, chain_id: wf_2]
     → (готовит контент)

✅ Результат: все 3 работают на одну цепочку wf_2,
              знают что это параллельные части одного проекта
```

### Сценарий 4: Итеративная обработка

```
User ← Victoria (вернул на правки)
  ↓
Marina (видит что вернулось из Victoria)
  ├─ [from: victoria] (не от user!)
  ├─ Понимает: это правки, нужно переделать
  ├─ Переделывает
  └─ [from: marina]
  ↓
Victoria (получает переделанное)
  ├─ [from: marina] (опять Марина)
  ├─ Проверяет правки
  └─ [VERDICT: done]

✅ Результат: полная история итераций в контексте
```

## 💻 КОД: ОБРАБОТКА КОНТЕКСТА

### Когда агент получает сообщение

```python
# В agent.py или любом агенте

import message_handler

# Входящее сообщение может содержать контекст
incoming = "[from: marina] [chain_id: wf_123] текст сообщения"

# Извлекаем контекст
context = message_handler.extract_request_context(incoming)
# {
#   "from_agent": "marina",
#   "to_agent": None,
#   "chain_id": "wf_123"
# }

# Используем контекст в логике
if context["from_agent"] == "user":
    print("Это первоначальный запрос от пользователя")
elif context["from_agent"] == "marina":
    print("Это контент от Марины, я редактирую")
else:
    print(f"Это контент от {context['from_agent']}")

# Когда пишем ответ, добавляем контекст обратно
response = message_handler.build_agent_message(
    content="Мой ответ",
    from_agent="victoria",
    to_agent=context.get("to_agent"),
    chain_id=context.get("chain_id")
)
```

### Когда отправляем результат следующему агенту

```python
# В браузере/JavaScript

const result = await fetch("/api/result?job=" + job_id);
const data = await result.json();

// Если есть следующий агент
if (data.next_agent) {
    // Используем контекст цепочки для следующего вызова
    const nextPayload = {
        agent: data.next_agent,
        message: data.reply,
        from_agent: data.agent_key,      // ← текущий становится отправителем
        to_agent: data.chain_context?.original_to_agent,
        chain_id: data.chain_context?.chain_id
    };
    
    // Отправляем следующему с полным контекстом
    await fetch("/api/chat", {
        method: "POST",
        body: JSON.stringify(nextPayload)
    });
}
```

## 🎯 ГЛАВНОЕ ПРЕИМУЩЕСТВО

Раньше:
```
User → Marina → Victoria → Vasya

Каждый знает только:
- Что он получил (текст)
- Что он должен сделать (обработать)
❌ НЕ знает откуда это пришло
❌ НЕ знает куда это идёт
```

Теперь:
```
User → Marina → Victoria → Vasya
     ↑         ↑        ↑
  [from]    [from]   [from]
  
Каждый знает:
✓ От КОГО пришло
✓ Какая ЦЕПОЧКА
✓ КОМУ это адресовано (если нужно)
✓ Может принимать КОНТЕКСТНЫЕ решения
```
