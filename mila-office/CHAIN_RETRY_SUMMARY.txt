KEY FEATURES
================================================================================

1. RETRY_CHAIN(chain_id, failed_agent, reason, max_retries=3)
   ✓ Restart from failed agent
   ✓ Track retry count
   ✓ Reset downstream agents
   ✓ Log to error_monitor
   ✓ Send Telegram alerts on critical failure
   ✓ Configurable max_retries per call

2. ESCALATE_CHAIN(chain_id, new_agent, reason="")
   ✓ Switch to different agent (for stuck/complex tasks)
   ✓ Skip remaining agents in original chain
   ✓ Update agents list dynamically
   ✓ Full audit trail

3. SPLIT_CHAIN(chain_id, to_agents[], context=None)
   ✓ Send task to multiple agents in parallel
   ✓ Track each branch independently
   ✓ Store branch metadata

4. MERGE_RESULTS(chain_id, results[], merge_strategy)
   ✓ "union" — Collect all successful results
   ✓ "consensus" — Pick highest confidence
   ✓ "first_success" — First non-error result
   ✓ Handle errors gracefully

EXAMPLE 1: Basic Sequential Chain
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
from chain_retry import create_chain, update_node_status, retry_chain, escalate_chain

# Create
chain = create_chain("content_post_001", ["victoria", "vasya", "olya"])

# Victoria edits
update_node_status("content_post_001", "victoria", "running")
response = run_agent("victoria", context)
update_node_status("content_post_001", "victoria", "success", reply=response)

# Vasya schedules (timeout!)
update_node_status("content_post_001", "vasya", "running")
try:
    response = run_agent("vasya", context)
    update_node_status("content_post_001", "vasya", "success", reply=response)
except TimeoutError as e:
    update_node_status("content_post_001", "vasya", "failed", error=str(e))

    # Retry Vasya
    result = retry_chain("content_post_001", "vasya", "timeout", max_retries=2)
    if result is None:
        # Max retries exceeded
        escalate_chain("content_post_001", "manager", reason="Vasya failed after retries")


EXAMPLE 2: Parallel Analysis with Merge
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
from chain_retry import split_chain, merge_results, complete_chain

# Split to 3 agents
split_chain("report_001", ["olya", "rita", "manager"])

# Run all agents
results = []
for agent in ["olya", "rita", "manager"]:
    try:
        response = run_agent(agent, {"month": "June"})
        results.append({
            "agent_key": agent,
            "result": response,
            "confidence": 0.95,
            "error": None
        })
    except Exception as e:
        results.append({
            "agent_key": agent,
            "result": None,
            "error": str(e),
            "confidence": None
        })

# Merge with consensus strategy
merge_results("report_001", results, merge_strategy="consensus")

# Complete
complete_chain("report_001", final_result="Report ready for review")

SUPPORT & TROUBLESHOOTING
================================================================================

Q: Chain not found?
A: Call create_chain() first, or check chain_id spelling

Q: Retries not working?
A: Verify error is retryable (check should_retry logic in integration)

Q: Tests fail?
A: Ensure no duplicate chain_ids; tests use unique IDs

Q: High memory usage?
A: Call clear_old_chains(hours=24) periodically

Q: Need chain history?
A: Call get_chain_history(chain_id) or check logs/chain_events.jsonl

Q: Want to export for analysis?
A: Use export_chain_to_json(chain_id) or monitor.export_failed_chains()
