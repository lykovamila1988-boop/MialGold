# 💡 CONTEXT EXAMPLES — Примеры использования контекста в агентах

## 🎯 Как использовать контекст в handle()

### Victoria (Редактор) — Пример 1: Понимание откуда пришло

```python
# victoria.py handle() обновленный:

def handle(name, inp):
    # Контекст может быть в системном prompt или в сообщении
    # Извлекаем контекст если он есть
    
    if name == "analyze_text":
        text = inp.get("text", "")
        
        # Если это редактура от Марины, то это уже готовый пост
        # Если это редактура от пользователя, то это может быть черновик
        from system_prompt_builder import extract_context_from_message
        
        # В реальном коде контекст будет в system prompt,
        # поэтому просто используем его в логике
        
        # Пример: если контекст указывает что это от marina,
        # то мы знаем что это пост готовый и нужна финальная проверка
        return f"✓ Текст проверен (пост от маркетолога)"
```

### Vasya (Планировщик) — Пример 3: Контекст цепочки помогает планировать

```python
# vasya.py - знает что это финальный результат перед публикацией

"""ТВОЯ ЗАДАЧА (обновленная):

"""

def handle(name, inp):
    if name == "schedule_post":
        post_id = inp.get("post_id", "")
        
        # Если это из цепочки, сохраняем chain_id для отслеживания
        from system_prompt_builder import extract_context_from_message
        
        # context будет доступен из system prompt
        # Это позволит нам отслеживать всю цепочку
        
        return f"✓ Пост {post_id} расписан и отслеживается"
```

## 💬 Как читать контекст в system prompt

### Для Victoria (редактор):

```
Ты видишь в system prompt строки типа:

✓ Ты получила запрос от: marina
✓ ID цепочки обработки: post_2026_06_08_1
✓ Следующий агент: vasya

Это означает:
- Марина отправила тебе текст → это должна быть редактура постового контента
- Цепочка отслеживается ID post_2026_06_08_1 → сохраняй этот ID
- После тебя идет vasya → подготавливай пост под расписание

Ты можешь даже написать в ответе что-то вроде:
"Пост готовый, отправляю Васе на расписание... [→ vasya]"
```

### Для Vasya (планировщик):

```
Ты видишь в system prompt:

✓ Ты получила запрос от: victoria
✓ ID цепочки обработки: post_2026_06_08_1
✓ ТЫ ПОСЛЕДНЯЯ В ЦЕПОЧКЕ — это финальный результат

Это означает:
- Виктория отправила готовый отредактированный пост
- Цепочка заканчивается у тебя → это финальная работа
- Проверь качество и расписывай спокойно

Ты можешь ответить:
"Пост отредактирован Victoria, расписываю на 13:00 по московскому времени. 
Трекинг: post_2026_06_08_1 [VERDICT: done]"
```

## ✨ ИТОГОВЫЙ АЛГОРИТМ

### Когда агент получает сообщение:

1. **Читает контекст** из system prompt:
   ```
   [from: ???] [to: ???] [chain_id: ???]
   ```

2. **Понимает роль**:
   - Первый ли в цепочке (from: user)?
   - Последний ли (is_final)?
   - Кому адресовано?

3. **Принимает решение**:
   - Что искупить в результате?
   - Кому передать?
   - Какой [VERDICT]?

4. **Пишет ответ** с контекстом:
   ```
   Мой ответ...
   [VERDICT: ready_next] [→ next_agent]
   ```

5. **Система** автоматически:
   - Переключается на следующего агента
   - Передает контекст дальше
   - Логирует цепочку
   - Отслеживает работу
