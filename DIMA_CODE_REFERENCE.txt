3. calculate_chain_roi(chain_id: str) -> str
   Location: mila-office/dima.py, lines 170-230

   Purpose: Calculate ROI with financial context and decision recommendation

   Input:  chain_id (string)

   Output: JSON string with:
     {
       "status": "ok" | "no_expenses",
       "chain_id": <input>,
       "total_expense_cad": <float>,
       "total_revenue_cad": <float>,
       "net_profit_cad": <float>,
       "roi_percent": <float>,
       "verdict": <str>,           # One of 4 verdicts below
       "financial_decision": <str>,
       "recommendation": <str>
     }

   Algorithm:
     1. Call get_expenses_by_chain() and get_revenue_by_chain()
     2. Parse both JSON responses
     3. Extract total_expense_cad and total_revenue_cad
     4. Calculate net_profit = revenue - expense
     5. Calculate ROI% = (profit / expense) × 100
     6. Assign verdict based on ROI:
        - ROI >= 300%   → "✓ Отличная цепочка — инвестируй больше"
        - ROI 100-299%  → "✓ Хорошая цепочка — продолжай в том же духе"
        - ROI 0-99%     → "⚠ Слабая цепочка — нужна оптимизация"
        - ROI < 0%      → "✗ Убыточная цепочка — срочные правки"
     7. Assign decision based on verdict
     8. Return full context with recommendation

   Financial decision mapping:
     Excellent (ROI >= 300%)  → Decision: "Увеличь бюджет"
     Good (100-299%)          → Decision: "Поддерживай уровень"
     Weak (0-99%)             → Decision: "Сократи расходы ИЛИ увеличь выход"
     Losing (< 0%)            → Decision: "Пересмотри стратегию или заморозь"


4. get_all_chains_summary() -> str
   Location: mila-office/dima.py, lines 233-297

   Purpose: Get overview of all chains sorted by ROI (worst first)

   Input:  (none)

   Output: JSON string with:
     {
       "status": "ok" | "no_data",
       "total_chains": <int>,
       "total_expense_cad": <float>,
       "total_revenue_cad": <float>,
       "total_net_profit_cad": <float>,
       "chains_by_roi": {
         "<chain_id_1>": {
           "expense_cad": <float>,
           "revenue_cad": <float>,
           "net_profit_cad": <float>,
           "roi_percent": <float>,
           "status": "✓ отлично|✓ хорошо|⚠ слабо|✗ убыток"
         },
         "<chain_id_2>": { ... },
         ...
       },
       "action_items": [
         "СРОЧНО: <chain_id> убыточна (ROI <roi>%)" | "Все цепочки в прибыли ✓"
       ]
     }

   Algorithm:
     1. Glob all expenses_*.json files in analytics dir
     2. For each file:
        a. Extract chain_id from filename
        b. Read expenses, sum total_expense
        c. Check if revenue_{chain_id}.json exists
        d. If yes, read revenue and sum total_revenue
        e. Calculate net_profit and roi_percent
        f. Map to status string
     3. Build chains dict with all calculations
     4. Sort by ROI ascending (worst first)
     5. Generate action_items (highlight negative ROI chains)
     6. Return complete summary

   Sorting: Ascending ROI (negative ROI first → warning sign)
   Action items: Auto-generate list of urgent problems

def handle(name, inp):
    if name == "gumroad_sales": return get_gumroad_sales(limit=20)
    if name == "calc_ltv_and_mrr": return calc_ltv_and_mrr()
    if name == "automation_stats": return automation_stats(inp.get("days", 7))
    if name == "get_purchases_data": return get_purchases_data(inp.get("days", 30))
    if name == "get_consultations_data": return get_consultations_data(inp.get("days", 30))
    if name == "check_supabase_access": return check_supabase_access()
    # NEW CHAIN_ID TOOLS:
    if name == "get_expenses_by_chain": return get_expenses_by_chain(inp.get("chain_id"))
    if name == "get_revenue_by_chain": return get_revenue_by_chain(inp.get("chain_id"))
    if name == "calculate_chain_roi": return calculate_chain_roi(inp.get("chain_id"))
    if name == "get_all_chains_summary": return get_all_chains_summary()
    # Fallback to core_handle for file operations
    res = core_handle(name, inp, list_default="05-analytics")
    return res if res is not None else f"Неизвестный инструмент: {name}"

Key pattern:
  - Each tool name has a handler that extracts input params and calls function
  - For chain-based tools: inp.get("chain_id") extracts the parameter
  - Return value is always a string (JSON or error message)

DATA STRUCTURE DETAILS
================================================================================

File path pattern: MILA-BUSINESS/05-analytics/expenses_<chain_id>.json

Expenses file format:
[
  {
    "date": "2026-06-08",              # When expense occurred
    "category": "design",               # Type of expense
    "description": "Figma макет",       # What was paid for
    "amount_cad": 15.00,                # Cost in CAD
    "contractor": "дизайнер"            # Who performed the work
  }
]

Revenue file format: MILA-BUSINESS/05-analytics/revenue_<chain_id>.json
[
  {
    "date": "2026-06-09",               # When purchase occurred
    "source": "praktikum_purchase",     # Type of transaction
    "customer": "customer_1",            # Customer identifier
    "amount_cad": 37.00,                # Revenue in CAD
    "note": "куплен после просмотра"   # Context (why bought)
  }
]

Key design choices:
  - Both files are JSON arrays (easy to parse)
  - Amounts in CAD (consistent with product pricing)
  - Dates in ISO format (YYYY-MM-DD, sortable)
  - Categories are free-text (flexible for different expenses)
  - Notes include customer journey context (helps understand conversions)

================================================================================
ROI CALCULATION FORMULA
================================================================================

net_profit = total_revenue - total_expense

roi_percent = (net_profit / total_expense) × 100

Examples:

  Expense: $100, Revenue: $400
  Profit: $300
  ROI: (300/100) × 100 = 300%
  Interpretation: Every $1 spent returned $3 profit

  Expense: $120, Revenue: $314
  Profit: $194
  ROI: (194/120) × 100 = 161.7%
  Interpretation: Every $1 spent returned $1.62 profit

  Expense: $275, Revenue: $614
  Profit: $339
  ROI: (339/275) × 100 = 123.3%
  Interpretation: Every $1 spent returned $1.23 profit

  Expense: $100, Revenue: $0
  Profit: -$100
  ROI: (-100/100) × 100 = -100%
  Interpretation: Lost $1 for every $1 spent (kill immediately)

================================================================================
COMMON QUESTIONS
================================================================================

Q: How are customer purchases attributed to a chain?
A: Manually - when recording a sale, note which content/campaign led to it.
   Example: "Практикум куплен после просмотра поста" in revenue file.

Q: Can a customer be in multiple chains?
A: Yes - customer might see reel AND click Instagram ad THEN buy.
   Attribute the purchase to the most direct/recent touchpoint.

Q: What if revenue can't be attributed?
A: Create a "direct" chain to track organic/unknown-source sales.

Q: How often should I update revenue files?
A: Ideally daily (as sales come in). Minimum weekly.

Q: What if expense is shared across chains?
A: Split it proportionally (e.g., video editor spends 40% on reels, 60% on ads).
   Or create a separate "shared-costs" chain.

Q: How long to wait for ROI to stabilize?
A: Content: 2-4 weeks (posts are evergreen)
   Ads: 1-2 weeks (usually immediate conversion)
   Email: 3-4 weeks (delayed opens/conversions)

Q: What's a "good" ROI?
A: For content: 300%+ (organic efficiency)
   For paid ads: 150%+ (must cover ad platform margins)
   For anything: > 0% (at least profitable)
